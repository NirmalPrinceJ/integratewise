Canonical ID Strategy

Identity Resolution
external_id

Original CRM identifier (SF Account ID, HubSpot Company ID)

→
crm_id

Source system reference with prefix (sf:, hs:, dyn:)

→
spine_id

Canonical UUID in Atlas Spine (immutable golden record)

Merge & Dedup Rules

  • Domain match: Email domain + company name fuzzy match (≥85% confidence)
  • Explicit link: Parent-child relationships via CRM hierarchy
  • Uniqueness: One spine_id per logical entity; soft-delete on merge
  • Audit trail: All merges logged with source IDs and timestamp
