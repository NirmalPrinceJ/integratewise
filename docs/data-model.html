Data Model

Canonical Data Model

Complete specification of all entities, relationships, keys, types, and constraints.

Core Entities

The canonical model normalizes data from all sources into these core entities.

accounts

Field Type Constraints Description
id UUID PRIMARY KEY Unique identifier
name VARCHAR(255) NOT NULL Account name
domain VARCHAR(255) UNIQUE Primary domain
health_score DECIMAL(5,2) CHECK (0-100) Current health score
created_at TIMESTAMP NOT NULL Record creation time

contacts

Field Type Constraints Description
id UUID PRIMARY KEY Unique identifier
account_id UUID FOREIGN KEY → accounts.id Associated account
email VARCHAR(255) NOT NULL, UNIQUE Contact email
role VARCHAR(100) NULL Job title/role

Additional Core Entities

The complete model also includes:

  • stakeholders - Key decision makers and influencers
  • integrations - Connected systems and adapters
  • workflows - Automated processes and agent configurations
  • runs - Execution logs and results
  • events - Time-series event data
  • secrets_vault - Encrypted credential storage
  • health_scores - Historical health score snapshots

For the complete schema specification, see IntegrateWise_Schema_v1.json.
