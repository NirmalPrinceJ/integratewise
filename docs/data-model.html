Technical

Canonical data model

Names, keys, types, and constraints for the customer hub. All tables are versioned; foreign keys cascade on delete unless noted.

Entities

accounts

id (uuid, pk), external_id (text, unique), name (text, not null), status (enum: active, churned, prospect), mrr (numeric, default 0), created_at (timestamptz, not null), updated_at (timestamptz, not null).

contacts

id (uuid, pk), account_id (uuid fk→accounts.id), email (text, unique), name (text), role (text), created_at, updated_at.

stakeholders

id (uuid, pk), account_id (uuid fk→accounts.id), contact_id (uuid fk→contacts.id), influence (enum: sponsor, champion, blocker), notes (jsonb), updated_at.

integrations

id (uuid, pk), type (enum: salesforce, hubspot, dynamics, zoho, custom), status (enum: active, paused), last_synced_at (timestamptz), secrets_vault_id (uuid fk→secrets_vault.id).

workflows

id (uuid, pk), name (text), description (text), trigger (jsonb), target_workspace (enum: notion, airtable, asana, clickup, monday, sheets), enabled (bool default true), created_at.

runs

id (uuid, pk), workflow_id (uuid fk→workflows.id), status (enum: queued, running, succeeded, failed), started_at (timestamptz), completed_at (timestamptz), attempts (int default 0), logs (jsonb).

events

id (uuid, pk), run_id (uuid fk→runs.id null on delete cascade), type (text), payload (jsonb), occurred_at (timestamptz).

secrets_vault

id (uuid, pk), name (text unique), rotated_at (timestamptz), rotation_policy_days (int), rbac_role (text), checksum (text), audit_ref (text).

health_scores

id (uuid, pk), account_id (uuid fk→accounts.id), model_version (text), score (numeric 0-100), drivers (jsonb), computed_at (timestamptz), ttl_minutes (int default 120).

Constraints and behaviors

  • All tables include created_at/updated_at triggers with clock skew tolerance ±60s.
  • health_scores rows expire when computed_at + ttl_minutes is exceeded; downstream views receive last-known-good.
  • runs enforce max attempts (5) with exponential backoff (2^n seconds) and jitter.
  • secrets_vault rotation enforced via rotation_policy_days; non-compliance raises VaultGuard alerts.
  • events payloads are immutable; redactions use checksum field on secrets_vault.
