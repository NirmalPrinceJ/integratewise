Architecture

Deployment and runtime

Atlas Spine

Core services deployed on Atlas Spine with stateless workers; adapters isolated per integration with secrets_vault references.

Health compute window

Health scoring windows run every 6 hours with drift detection; out-of-window runs queued with priority to prevent backlogs.

Retry and backoff

Runs use exponential backoff (2^n seconds, max 5 attempts) with jitter and dead-letter after final failure; observability via runs/events tables.

Data residency

Primary region: India; optional EU/US deployment via separate Atlas Spine clusters and dedicated secrets_vault entries.
